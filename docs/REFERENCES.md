# External References

This document lists the key external resources used (or worth knowing about) for
the PT3 player implementation. Keep this updated when encountering new useful links.

---

## Primary reference: Vortex Tracker II

**The authoritative PT3 implementation** — our Python simulator is a line-by-line port
of its playback engine, and all our PSG test files were generated by this tool.

- **Author**: Ivan Roshin (VBsoft)
- **Key source file**: `trfuncs.pas` — 10,810 lines of Turbo Pascal, contains the full
  PT3 decode + playback engine. Every design note in our `pt3_python_sim/README.md`
  references specific line numbers here.
- **Why it matters**: VTII is the current de-facto standard for PT3 playback and file
  generation. Matching VTII bit-exact = matching what modern trackers and players
  produce. When VTII and Vince Weaver's `README_pt3.txt` spec disagree, VTII wins
  (see our own 14 design notes).

## Bulba VTII10 r7 (Z80 asm)

Origin of the note frequency tables and volume combination tables.

- **Author**: S.V.Bulba
- **Content**: Z80 assembly source for the original ProTracker 3 player on ZX Spectrum
- **What we ported**: `NoteTableCreator` and `VolTableCreator` algorithms — these are
  reimplemented line-by-line in `pt3_tables.py`. The 6502 port should do the same
  (Python reference is small and trivially portable).
- **Note**: Bulba's asm player uses an "old" volume table for PT3 < 4. **VTII
  overrides this decision and uses the "new" table for all versions.** We follow
  VTII, not Bulba. See design note #2 in simulator README.

## ZXTune (Vitamin/CAIG)

**Cross-platform open-source chiptune player** covering ~50 different chip music
formats including PT3, PT2, AY, YM, PSG, VTX, and many Amiga/Atari/PC trackers.

- **Website**: https://zxtune.bitbucket.io/
- **Source (primary)**: https://bitbucket.org/zxtune/zxtune
- **Source (GitHub mirror, easier browsing)**: https://github.com/vitamin-caig/zxtune
- **License**: LGPL-3.0
- **Language**: C++ (70%) + Kotlin (Android app)
- **Size**: ~500k lines, 6200+ commits, maintained since 2008
- **PT3 parser location**: `src/formats/chiptune/aym/` (file decoder) and
  `src/module/players/aym/` (playback engine). Look for `ProTracker3*.cpp`.
- **Regression corpus**: `regression/` directory contains their bit-exact test files
  — potentially useful as additional ground truth if we ever find a PT3 file where
  VTII and ZXTune disagree.

**In retrospect** this is the one reference that would have saved significant
reverse-engineering time. Our Python simulator independently rediscovered most of
what's already encoded in ZXTune's C++ implementation. However:

- Our code is **~2100 lines Python** vs ZXTune's C++ implementation (PT3-only slice
  is probably a few thousand lines, embedded in much larger infrastructure).
- We have **extensive design notes** explaining WHY each edge case matters — ZXTune
  has working code, but reading C++ doesn't directly give you the reasoning.
- **Bit-exact against VTII** is the goal we proved; ZXTune also likely matches VTII,
  but we didn't need to rely on that transitive trust.

**When ZXTune could still help us**:
1. **Tie-breaker**: if we ever find a PT3 file where VTII-generated PSG disagrees with
   ZXTune-generated PSG (unlikely but possible for exotic edge cases), we have a
   third opinion.
2. **Consistency check**: if our 6502 port produces something strange on a specific
   file, comparing against ZXTune's C++ behavior gives independent validation.
3. **Other formats**: if we ever want to extend the Plus/4 player beyond PT3 (e.g.
   PT2, ASC, STC trackers from the same ZX era), ZXTune has them all implemented
   and is a natural reference.

**Do NOT port ZXTune's code directly**. It's C++ with heavy abstractions (factory
patterns, generic chip emulation layers) designed for a 50-format player. Our
target is a 4 KB standalone module for one format on a 7501 CPU. Different
constraints → different code.

## Vince Weaver's PT3 format spec

**`README_pt3.txt`** — the classical format documentation.

- **Useful for**: big-picture overview of the file layout, header fields, opcode
  ranges.
- **Not useful for**: bit-exact playback behavior. Several subtleties (opcode $10
  behavior, envelope shape numbering offset, PORTM parameter layout, noise_period
  persistence semantics) are either wrong, omitted, or ambiguous.
- **Rule of thumb**: when Vince says one thing and VTII does another, VTII wins.
  See design notes #3, #5 in the simulator README for concrete examples.

## AY-3-8910 / DigiMuz hardware

The target sound chip. All our PSG output is "what values go into which AY register
at what frame".

- **General Instruments AY-3-8910 datasheet**: `registersgeneralinstrument_ay38910.pdf`
  (in project root)
- **DigiMuz expansion card for Plus/4**: Peripheral at $FD21 (read), $FD22 (data
  write), $FD23 (register select). Register write sequence: `STX $FD23 / STA $FD22`
  — ~8 cycles separation exceeds AY's 400ns setup time, no NOPs needed.
- **World of Spectrum project AY page**: https://www.worldofspectrum.org/projectAY.html
  — repository of historical AY format files (note: this is `.ay` emulator-dump
  format, NOT PT3).

## TED / Plus/4 hardware

Target platform.

- **TED System Hardware Manual**: `TED_System_Hardware_Manual.pdf` (in project root)
- **SVS Ultimate Map v2.2**: `SVSUltimateMap_v22xlsm.xlsm` (in project root) —
  comprehensive memory map including safe zero-page regions, banking, TED
  registers, I/O area.
- **Relevant TED registers for our IRQ**:
  - Timer 1: $FF00 (lo), $FF01 (hi) — autoreload timer, source for playback IRQ
  - IRQ enable: $FF0A bit 5 (Timer 1)
  - IRQ status/ack: $FF09 bit 4 (Timer 1 pending)
- **IRQ rate**: 60 Hz NTSC = autoreload $3A43; 50 Hz PAL = $4523

## Test corpus

Our 20-file test corpus is stored in `pt3_python_sim/test_files/` (delivered as zip
to Kris). When the 6502 port is ready, we'll validate it against **the same 20 files**
using py65 as the emulator and our Python simulator output as the oracle.

Test file source: a mix of scene party releases (2025), historical game soundtracks
(1988, 1990), and early demo releases (1998, 2003). Covers all PT3 versions 3.3
through VTII 1.0, all FeaturesLevels, all tone tables, all documented effects.
